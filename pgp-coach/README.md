# GTM Collective — PGP Coach v2

AI-powered Personal Growth Plan coaching app with modular sessions, save/resume, and admin portal.

## What's New in Phase 1
- **PIN login** — hires log in with a 6-digit PIN generated by admin
- **6 independent modules** — complete at your own pace, progress saves automatically
- **Admin portal** at `/admin` — create hires, manage tags, view completed plans
- **Postgres database** — all conversations and plans stored persistently
- **Save & resume** — close the browser, come back later, pick up exactly where you left off

---

## Deploying on Railway (recommended)

### Step 1 — Add a Postgres database
1. In your Railway project, click **+ New** → **Database** → **PostgreSQL**
2. Railway automatically adds `DATABASE_URL` to your environment — you don't need to copy anything

### Step 2 — Add environment variables
In your Railway service → **Variables** tab, add:
```
ANTHROPIC_API_KEY=sk-ant-your-key-here
SESSION_SECRET=any-long-random-string-here
NODE_ENV=production
```

### Step 3 — Push this code to your GitHub repo
Replace the contents of your `pgp-coach` repo with these files, then push.
Railway will automatically redeploy.

### Step 4 — First login
- **Admin portal:** go to `your-url.railway.app/admin`
  - Email: `admin@teamrevenue.com`
  - Password: `admin123`
  - ⚠️ Change this password after first login (update the hash in schema.sql)
- **Hire app:** go to `your-url.railway.app`

---

## Running locally

```bash
# Set environment variables
export DATABASE_URL=postgresql://localhost/pgp_coach
export ANTHROPIC_API_KEY=sk-ant-your-key
export SESSION_SECRET=local-dev-secret

# Install and run
npm install
node server.js
```

## File structure
```
├── server.js              # Express server + session handling
├── db/
│   ├── index.js           # Postgres connection pool
│   └── schema.sql         # Database tables (auto-runs on startup)
├── routes/
│   ├── auth.js            # Login/logout for admins and hires
│   ├── admin.js           # Admin API (hires, tags, plans)
│   └── hire.js            # Hire API (dashboard, modules, chat)
└── public/
    ├── index.html         # Hire-facing app (PIN login + modules + chat)
    └── admin.html         # Admin portal
```
